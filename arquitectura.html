Servicios del stack

Open WebUI Frontend

Interfaz de chat web donde el equipo hace preguntas en lenguaje natural. Se conecta a LiteLLM mediante API compatible con OpenAI.

LiteLLM Gateway

Router de modelos IA. Expone una API unificada y enruta las peticiones de chat hacia OpenRouter. Gestiona modelos, keys y logs en Postgres.

Qdrant Vector DB

Base de datos vectorial self-hosted. Almacena embeddings de código (1024 dims, distancia coseno) con metadata enriquecida: repo, rama, lenguaje, entidad AST.

Neo4j Graph DB

Base de datos de grafo (Community 5.26 + APOC). Nodos CodeEntity y relaciones: EXTENDS, IMPLEMENTS, HAS_METHOD, HAS_FIELD, CALLS, INJECTED, ANNOTATED_WITH.

JavaParser Parser

Microservicio Javalin que parsea código Java/Spring Boot con JavaParser 3.26. Extrae anotaciones, herencia, interfaces, campos inyectados y llamadas a métodos. Fallback a Tree-sitter.

Indexador Backend

Servicio FastAPI en http://indexer:8001. Escucha webhooks de GitHub, clona repos, parsea AST, construye grafo Neo4j y genera embeddings vía OpenRouter.

Postgres Database

Base relacional que persiste configuración, logs y métricas de LiteLLM.

OpenRouter Proveedor IA

API externa que provee embeddings (qwen/qwen3-embedding-8b, 1024 dims) y modelos de chat (deepseek/deepseek-v4-pro principal, deepseek/deepseek-v4-flash fallback).

Integración RAG con Open WebUI

La función tennis_doc_rag.py actúa como un Filter dentro de Open WebUI. Intercepta cada conversación antes de enviarla al LLM:

Comando especial: escribe indexa TennisDev/mi-repo o indexa TennisDev/mi-repo develop en el chat y la función disparará POST /index automáticamente, sin necesidad de entrar al contenedor.

Ver grafo de una clase: escribe grafo NombreClase en el chat y la función consultará Neo4j para mostrar sus relaciones directas (herencia, implementaciones, métodos, campos, inyecciones).

Listar repos: escribe repos en el chat para ver todos los repositorios indexados.

Exportar a Markdown: escribe generame un md con toda la info del módulo X, guardar como markdown o exportar a md. Si el contenido es pequeño se muestra en el chat; si es grande se genera un archivo descargable.

Activación en Dokploy / Docker Compose

La función ya viene embebida dentro del docker-compose.yml como un config inline. Open WebUI la recibe automáticamente al iniciar. Solo debes ir a Admin Panel → Functions, buscar Tennis Doc RAG y activar el toggle.

Haz clic en ⚙️ Config de la función para cambiar la rama por defecto (prod) o ajustar la cantidad de chunks.

Importante: el archivo tennis_doc_rag.py en el repo y la versión embebida en docker-compose.yml deben mantenerse sincronizados.

Mejoras recientes

🔐 Seguridad: API Key en endpoints

Variable INDEXER_API_KEY protege todos los endpoints públicos del indexer con Authorization: Bearer. El webhook sigue usando HMAC-SHA256.

🧠 Arquitectura v2.0: AST + Grafo + LlamaIndex

Parseo AST profesional con Tree-sitter y JavaParser. Neo4j almacena el grafo de dependencias. LlamaIndex orquesta retrieval híbrido.

🕸️ Grafo de dependencias

Neo4j indexa relaciones tipadas: EXTENDS, IMPLEMENTS, HAS_METHOD, HAS_FIELD, CALLS, INJECTED, ANNOTATED_WITH. Endpoint /graph/entity/{name} para consultar jerarquías.

🔍 Retrieval híbrido mejorado

El filtro primero consulta /search-augmented (archivos completos), luego /search-graph (relaciones), y trae entidades del grafo si se mencionan nombres de clase. Límites aumentados: 50 vectoriales, 10 archivos, 20 resultados finales.

🗂️ Listado de repos

Nuevo endpoint GET /repos y comando repos en el chat para listar todos los repositorios indexados.

📎 SQL Inline Resolver (truncado)

SQL referenciado en Java se incrusta inline truncado a 6.000 chars. El archivo .sql completo sigue indexándose por separado, evitando chunks gigantes.

📄 Markdown mejorado

Regex más permisivo (guardar como md, exportar a markdown). Si el contenido es < 8.000 chars se muestra en el chat; si es grande se genera descarga.

🧱 Mitigación de clases monstruo

Clases/interfaces con > 12.000 chars se truncan en Qdrant. El embed_text limita código a 6.000 chars y relaciones a 20, evitando truncamiento del embedding.

🔑 Auto-renew GitHub token

Token de instalación se renueva automáticamente al detectar 401 Unauthorized durante indexaciones largas.

🔄 Reintentos automáticos

El embedder reintenta hasta 3 veces con backoff exponencial ante fallos de red o rate-limits de OpenRouter.

🏥 Health check real

GET /health verifica conectividad con Qdrant, Neo4j y JavaParser antes de reportar estado saludable.

🔍 Endpoints de diagnóstico

/debug/files-indexed, /debug/chunks, /graph/entity/{name}, /graph/related/{entity_id} y /repos permiten inspeccionar índices y grafo.

⚙️ Configuración centralizada

Módulo config.py centraliza variables de entorno; docker-compose.yml usa named volumes para persistencia.
